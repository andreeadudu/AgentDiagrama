# 📐 Diagrama — AI Diagram Generation Agent

Diagrama is an AI agent that turns a plain-text description of a system into a diagram. You describe how something works — a login flow, a database structure, a set of microservices talking to each other — and the agent writes back a diagram you can actually use in documentation.

It's built for software architects: instead of manually drawing every diagram by hand, you describe the system once and get a valid, ready-to-use diagram back in seconds.

**Example:**

````
You: Draw a sequence diagram for a user logging in through an
     auth service that checks a database

AI: ```mermaid
sequenceDiagram
    participant User
    participant AuthService
    participant Database
    User->>AuthService: POST /login
    AuthService->>Database: Check credentials
    Database-->>AuthService: User record
    AuthService-->>User: 200 OK + token
```
This shows the login flow between the user, the auth service,
and the database.
````

**Supported diagram types:** Flowchart, Sequence Diagram, Class Diagram, ER Diagram, State Diagram.

## How It Works

1. You type a description (or point to a `.txt` file with a design doc).
2. The agent searches its internal knowledge base for relevant Mermaid syntax rules, using a local embedding model.
3. It sends your request, plus that reference material, to a language model (Azure), which writes the diagram.
4. The diagram is automatically checked for correctness before being shown to you.
5. You can ask the agent to save it as a file.

Two different AI models are used for two different jobs: a small local model just for searching the knowledge base, and a larger cloud model for actually writing the diagram. This keeps the search step fast and cheap, while the diagram itself is generated by a more capable model.

## Installation

**You'll need:** Python 3.10+, [Ollama](https://ollama.com/download) installed locally, and an Azure AI Foundry API key.

```bash
git clone https://github.com/andreeadudu/AgentDiagrama.git
cd AgentDiagrama

pip install -r requirements.txt

ollama pull bge-m3

# Add your API key and endpoint in config.py

python embedding_generator.py
```

## Running It

**Command line:**
```bash
python main.py
```

**In a browser (Flask):**
```bash
python web_app.py
```
Then open `http://localhost:5000` — diagrams are rendered visually.

**In a browser (Streamlit):**
```bash
streamlit run streamlit_app.py
```

## What Makes It Reliable

- **It checks its own work** — every diagram is validated for correct syntax before you see it.
- **It doesn't break on errors** — if the search model, the language model, or a file is unavailable, the agent explains what happened and keeps the conversation going instead of crashing.
- **It manages cost** — old, no-longer-relevant search results are cleared out each turn, and very long conversations are trimmed automatically to avoid runaway token usage.
- **It's easy to extend** — adding a new capability (a new tool, or new reference material) takes one new file and one line of registration, with no changes to the core logic.

## Configuration

All settings live in `config.py` — chunk size, search thresholds, token limits, and pricing — nothing is hardcoded elsewhere in the project.
